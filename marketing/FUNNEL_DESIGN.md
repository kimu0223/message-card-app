### ボトルネック詳細

**[LEAK-1] LP→/create (59%離脱)**
- Hero が 320vh のシネマティックスクロール。CTA ボタンが最下部にしか出ない
- X から流入したユーザーはすでに「カードを作りたい」心理。説得コンテンツ不要
- モバイルで体験が壊れている可能性（高bounce rate: 65-83%）

**[LEAK-2] /create→/editor (64%離脱)**
- テンプレート選択 UI で迷いが発生していると推定
- 「どれを選べばいいかわからない」段階で離脱
- 選んだ後の次ステップが不明確

**[LEAK-3] /editor→signup (~100%離脱)**
- ゲスト制限: 30日に1回のAIデザイン生成のみ
- カードを完成させる前に「登録してください」の壁がある
- 価値体験ゼロで離脱

## 2. 新ファネル設計

### Stage 0: AWARENESS（X → LP）

**目的**: 「作りたい」という意欲を持つユーザーを引き込む

**X投稿タイプ別設計:**

| タイプ | 内容 | CTAリンク | 期待効果 |
|---|---|---|---|
| 完成カード動画 | 実際に動くアニメーションカード15秒動画 | `/create?template=[id]` | 直接テンプレートへ |
| ビフォーアフター | 「伝えにくい気持ち → カードにしたら」 | `/create` | 感情的共感 |
| シーズン訴求 | 「父の日まであと14日」 | `/create?occasion=father` | 締め切り緊張感 |
| How-to Thread | 「3分でできるカードの作り方」5ポスト | `/create` | 教育 + 信頼 |
| ユーザー体験 | 「実際に使った感想」引用RT | LP | 社会的証明 |

**LPリンク先の工夫:**
- X投稿から来たユーザー → `/?utm_source=x&utm_medium=organic` でトラッキング
- テンプレート紹介投稿 → `/create?template=xxx` で直接エディタへ

---

### Stage 1: CONSIDERATION（LP改善）

**現状の問題**: Hero が長すぎてCTAまで到達しない

**改善案:**

1. **Sticky CTA バー** (最優先)
   - LP上部に常時表示される「無料でカードを作る →」バー
   - スクロール位置に関わらず常に見える
   - 実装: `position: sticky; top: 0` or fixed header にCTA追加

2. **Heroの短縮** (中優先)
   - 320vh → 200vh に短縮
   - モバイルでは 150vh
   - 「スクロールしなくても分かる」構成に

3. **テンプレートギャラリーのCTA化**
   - ギャラリーセクションの各カード → クリックで直接 `/create/editor?template=xxx`
   - 「このデザインで作る」ボタンを各テンプレートに追加

4. **社会的証明の強化**
   - StatStrip の「38万通」は実際の数値に更新（現在フェイク値）
   - ユーザー体験談を実名（許可取り）で掲載

---

### Stage 2: ACTIVATION（/create → /editor改善）

**目的**: テンプレート選択の摩擦をゼロにする

**改善案:**

1. **/create ページの再設計**
   - 現状: テンプレート一覧（迷う）
   - 改善: 「どんなシーンですか？」3択ボタン（誕生日 / 記念日 / お礼）から始める
   - 選んだら該当テンプレート上位3つだけ表示 → 即エディタへ

2. **テンプレート選択→エディタの導線**
   - クリック → 確認不要で即 `/create/editor?template=xxx` へ遷移
   - 「選んだ後どうすればいいか」のナビゲーション追加

3. **プログレスバー表示**
   - ステップ1: テンプレート選択 → ステップ2: メッセージ入力 → ステップ3: 送る
   - ユーザーが今どこにいるか常に明示

---

### Stage 3: VALUE DELIVERY（エディタ内体験）

**目的**: 「使ってよかった」体験を登録前に届ける

**現状の問題**: ゲスト = AIデザイン30日に1回しか使えない

**改善案:**

1. **ゲスト制限の緩和（最重要）**
   - 現状: `GUEST_SESSION_LIMIT = 1` (30日1回)
   - 改善案: 1枚目は制限なし。2枚目から登録要求
   - 実装: `src/lib/guest-limit.ts` の `GUEST_SESSION_LIMIT` を上げる or 「完成まで制限なし」に変更

2. **価値体験完了後に登録促進**
   - カード完成 → アニメーションプレビュー表示
   - その後「保存・シェアするにはアカウント登録（無料）」
   - 体験後に登録要求することで転換率が大幅に改善

3. **登録ハードルの最小化**
   - Googleアカウント1クリック登録
   - 「メールアドレスすら不要」オプションを検討

---

### Stage 4: RETENTION & MONETIZATION

**目的**: 無料ユーザーを有料へ転換

**改善案:**

1. **バイラルループ設計**
   - カードを受け取った相手 → 「あなたもカードを作る？」CTA付きの受信ページ
   - シェアURL: `/{cardId}?ref=share` で流入トラッキング

2. **無料枠の「見える化」**
   - ダッシュボードに「今月の残り枚数: 2/3」を常時表示
   - 枠消費時に「クレジットを追加すれば今すぐ作れます」

3. **アップグレードトリガー設計**
   - トリガー1: 3枚目のカード作成時
   - トリガー2: AIデザイン2回目使用時
   - トリガー3: プレミアムテンプレートクリック時
   - 各トリガーで底値オファー: 「¥400で5クレジット」を先に見せる

4. **再来訪設計**
   - カードを送った相手が開封したら通知 → 再訪トリガー
   - シーズン前2週間にメール/プッシュ通知（父の日・敬老の日・バレンタイン等）

// テンプレート選択イベント
gtag('event', 'template_selected', { template_id: 'xxx', source: 'gallery' })

// エディタ完成イベント
gtag('event', 'card_completed', { is_guest: true, template_id: 'xxx' })

// 登録モーダル表示
gtag('event', 'signup_modal_shown', { trigger: 'card_completed' })

// 登録完了
gtag('event', 'sign_up', { method: 'google', trigger: 'card_completed' })

// クレジット購入
